\title{\textbf{Tennis Ball Inspector: A Classic Computer Vision Pipeline \\
for Automated Visual Defect Detection in Tennis Balls}}
\author{Yuval Rubin \\
\small Holon Instatute Of Technolegy}
\date{June, 2026}

\begin{document}

\begin{abstract}
\noindent
This is a classical (non-deep-learning) computer vision system for automated
visual quality inspection of tennis balls. Given a single reference (template)
image of a new, defect-free ball and a runtime inspection image of a candidate
ball, the system localizes the ball, corrects for rotation and scale, extracts
a set of perceptually meaningful regions of interest (the printed logo
characters, the felt patch, and the two seam lines), and compares each region
against the reference using classical color and structural similarity
measures. Defects are flagged automatically using user-configurable
thresholds on: ball radius ratio, felt color difference (Earth Mover's
Distance in CIELAB space), seam color difference, logo character color
difference, and logo character structural difference (binary mask XOR after
morphological filtering). The system was evaluated on five inspection images
exhibiting different degrees and types of wear,
compared against one reference image. Results show that the pipeline
reliably separates lightly-used balls from heavily-worn ones, correctly
localizes the artificial seam defect. The report includes quantitative results for all test imagesand outline the
limitations and possible extensions of the system.
\end{abstract}

\vspace{0.3cm}
\noindent\textbf{Keywords:} visual inspection, anomaly detection, classical
computer vision, image registration, hough transform, CIELAB color space,
Earth Mover's Distance, morphological filtering, histograms, OpenCV.

\section{Introduction}
% ======================================================
Automatic visual inspection - comparing an image of a manufactured or
used object against a reference image of the same object in good condition is a classical task in industrial machine vision. Such systems are used to
catch printing defects, surface damage, deformation, and color degradation
without manual human inspection. The task is interesting precisely because
the ``reference vs.\ inspection'' images are never pixel-aligned: the object
may be rotated, translated or scaled, so a meaningful comparison requires a registration stage before any difference can be measured.

This project implements such a system for a everyday object that exhibits
rich, naturally-occurring wear patterns on a \textbf{tennis ball}. A tennis
ball has several distinct, independently-checkable features:
\begin{itemize}[noitemsep]
    \item a roughly circular silhouette (easy to localize with a Hough
    transform algorithm).
    \item two seam lines that must remain horizontal and intact after
    registration.
    \item a printed multi-character logo whose individual characters can
    degrade or disappear with wear.
    \item a uniformly colored felt surface whose color fades with use.
\end{itemize}

The goal of the system is to take one reference image of a new ball and one
inspection image of a candidate ball, and automatically report, per
feature, whether the candidate is within tolerance of the reference --
along with a numeric confidence/difference score for each feature.

\section{Methods}

\subsection{Pipeline Overview}

The system follows a four-stage pipeline, illustrated schematically in
Figure~\ref{fig:block_diagram}: (1) preprocessing and registration, which
brings the inspection image into the same coordinate frame as the
reference; (2) region-of-interest (ROI) extraction; (3) feature extraction,
which converts each ROI into a compact numeric or histogram representation;
and (4) comparison and decision, which measures the distance between
reference and inspection features and thresholds each distance to produce a
pass/fail verdict.

A key design decision is that \textbf{all reference-image features are
computed once, when the system is initialized}, and cached. Every
subsequent inspection image is processed through the identical pipeline and
compared against this single cached reference, rather than recomputing the
reference each time.

\subsection{Stage 1: Preprocessing and Registration}

\paragraph{Ball localization.} The ball is localized using the Hough
Circle Transform (\texttt{cv2.HoughCircles}) applied to a Gaussian-blurred
grayscale version of the image. The algorithm first applies edge-detection
operators (gradient-based kernels) to identify strong intensity transitions in
the image, which correspond to potential object boundaries. These edge points
are then used in a voting scheme where each point contributes votes in a
parameter space of possible circle centers and radii. The accumulator collects
these votes across all candidates, and the highest-scoring peak is selected as
the most likely hypothesis for the ball, yielding its center and radius
(Figure~\ref{fig:hough}).

\begin{figure}[H]
\centering
\includegraphics[width=0.55\textwidth]{figs/hough_circle_example.png}
\caption{Hough Circle Transform output on the reference image. The detected
circle (green) is used to center the subsequent square crop.}
\label{fig:hough}
\end{figure}

\paragraph{Cropping.} A fixed-size square region centered on the detected
ball is cropped from the original image (zero-padding is applied if the
crop would exceed the image boundary). This produces a consistent
coordinate frame in which all subsequent ROI coordinates (expressed as
fractions of image width/height in the configuration file) are valid
regardless of the ball's original position or scale in the source photo.

\paragraph{Rotation correction (registration).} Tennis balls can be
photographed at an arbitrary rotation. To register
the inspection image to the reference orientation, the two felt seam lines
are detected by thresholding the saturation channel of the HSV color space
(the white/gray seams have markedly lower saturation than the surrounding
yellow felt).

To improve robustness, the binary saturation mask is first smoothed using a
Gaussian filter and then refined through morphological opening with a
strongly anisotropic horizontal structuring element (a $50 \times 2$
rectangular kernel). This kernel suppresses isolated noise and preserves
elongated horizontal structures corresponding to seam lines, while removing
small vertical or blob-like artifacts introduced by the fuzzy felt texture (see Figure~\ref{fig:seammask}).

A best-fit line is computed through the remaining seam-line pixels
(\texttt{cv2.fitLine}), giving an estimated rotation angle for the upper and
lower seam independently; the two estimates are averaged and the whole image
(plus both seam masks) is rotated by the negative of this angle so that both
seams become approximately horizontal.

\begin{figure}[H]
\centering
\includegraphics[width=0.45\textwidth]{figs/seam_line_bin.png}
\caption{Binary mask obtained after saturation thresholding and refinement using a horizontal morphological kernel.}
\label{fig:seammask}
\end{figure}

\subsection{Stage 2: Region-of-Interest Extraction}

After registration, three ROIs are extracted from fixed, configurable
fractional coordinates of the registered crop:

\begin{itemize}
\item \textbf{Felt patch ROI}: a rectangular strip of plain felt (away
from the seams and logo), used to characterize the overall ball color.
\item \textbf{Seam ROIs}: the upper and lower seam-line search bands
(also reused from the registration step).
\item \textbf{Logo ROI}: a central rectangular band thresholded (dark-ink
binary mask via grayscale thresholding) and cleaned with a morphological
\emph{closing} operation (a vertical structuring element bridges small
gaps within each printed character - caused by the ball felt).
\end{itemize}

\paragraph{Character segmentation.} The cleaned logo binary mask
(Figure~\ref{fig:logomask}) is segmented into individual connected
components (\texttt{cv2.connectedComponentsWithStats}); components smaller
than an area threshold are discarded as noise. The remaining components
are sorted by row, then by horizontal position within each row, giving a
consistent left-to-right, top-to-bottom ordering of characters that is
comparable between the reference and any inspection image -- \emph{provided
the same number of characters is detected in both}. This character count
agreement is itself reported as a binary pass/fail feature, since a
mismatch indicates the logo has degraded enough that letters have merged,
disappeared, or split into fragments.

\begin{figure}[H]
\centering
\includegraphics[width=0.45\textwidth]{figs/logo_bin_mask.png}
\caption{Cleaned binary logo mask after grayscale thresholding and
morphological closing, ready for connected-component character
segmentation.}
\label{fig:logomask}
\end{figure}

\subsection{Stage 3: Feature Extraction}

\paragraph{Color signatures.} For each color-relevant ROI (patch, upper
seam, lower seam, and each logo character), the system computes a 2D color
histogram in CIELAB space over the Lightness ($L^{*}$) and Blue/Yellow
($b^{*}$) channels (100 $\times$ 100 bins, $L^{1}$-normalized). CIELAB was
chosen over RGB because its $L^{*}$ and $b^{*}$ axes more directly capture
the two failure modes of interest in this object -- felt darkening/fading
(lightness) and yellow-to-green color shift (blue/yellow axis) -- while
being more perceptually uniform than raw RGB differences. This 2D
histogram is referred to in the code and in this report as the ROI's
\emph{color signature}.

\paragraph{Structural (shape) signatures.} For each segmented logo
character, the raw binary mask is kept (cropped to its bounding box) as the
character's structural signature. To compare two characters' shapes, the
inspection character's mask is first translated (not rotated or scaled) so
that its centroid aligns with the corresponding reference character's
centroid -- this assumes residual misalignment after Stage~1 is small and
purely translational, which held in practice for our dataset.

\paragraph{Radius.} The Hough-detected radius of the inspection ball is
retained as a single scalar feature.

\subsection{Stage 4: Comparison and Decision}

\paragraph{Color difference -- Earth Mover's Distance.} Two color
signatures (reference vs. inspection, for the same ROI) are compared using
the Earth Mover's Distance, EMD.
computed via \texttt{cv2.EMD} on a sparse signature representation of each
histogram. EMD was chosen over a simpler bin-wise distance (e.g Bhattacharyya) because it is robust to small shifts of the
color distribution -- relevant here, since felt fading shifts the entire
color cluster rather than redistributing mass within fixed bins. The raw
EMD value is normalized by the maximum possible distance between any two
bins ($\sqrt{2} \cdot (\text{bins}-1)$) and clipped to $[0, 1]$, giving a
ROI color difference score that is comparable across ROIs and images.

\paragraph{Structural difference -- masked XOR.} Two aligned character
masks are compared with a pixel-wise logical XOR, producing a raw
difference map. This map is then passed through a morphological
\emph{erosion} with a $12\times12$ kernel: this acts as a noise filter,
removing thin, isolated XOR-difference pixels caused by minor edge jitter
from segmentation/alignment while preserving larger, contiguous regions of
genuine shape disagreement. The remaining nonzero pixel count, normalized by mask
area, is the character's structural difference score.

\paragraph{Thresholding.} Table~\ref{tab:thresholds} lists the final
decision thresholds, set empirically by the authors after inspecting score
distributions on the reference image and the worn-ball examples.

\begin{table}[H]
\centering
\caption{Decision thresholds for "TRETORN SERIE + CONTROL" balls (from
\texttt{config/tretorn\_serie\_plus\_control.json}).}
\label{tab:thresholds}
\begin{tabular}{@{}lll@{}}
\toprule
\textbf{Feature} & \textbf{Threshold} & \textbf{Pass condition} \\
\midrule
Ball radius ratio & $[0.95,\ 1.05]$ & ratio within range \\
Felt patch color difference (EMD) & $0.02$ & difference $\le$ threshold \\
Seam color difference (EMD, avg.\ of both seams) & $0.04$ & difference $\le$ threshold \\
Logo character color difference (EMD), per character & $0.035$ & difference $\le$ threshold \\

\bottomrule
\end{tabular}
\end{table}

All thresholds, ROI coordinates, and the ball radius search range are
externalized to a single JSON configuration file, so the system can be
re-tuned for a different ball model (e.g.\ a different logo or felt color)
without code changes.

\subsection{Implementation}

The system is implemented in Python 3 using OpenCV for all image
processing and NumPy for array operations; Matplotlib is used only for the
optional visualization dashboard, not for any inspection decision. The
codebase is organized into four modules (\texttt{Preprocessor},
\texttt{ColorExtractor}, \texttt{FeatureExtractor}, \texttt{BallInspector})
described in full in Appendix~A, with the orchestration logic in
\texttt{ball\_inspector.py}. No machine-learning models, training data, or
deep-learning libraries are used anywhere in the pipeline; every step is a
classical image-processing operation (thresholding, Hough transform,
connected components, morphological filtering, histogram comparison).

\section{Discussion and Conclusions}
% ======================================================

The pipeline separates clearly-defective tennis balls (heavy wear,
artificially marked seams, recolored or filled logo characters) from a
clean reference using only classical image processing, with no learned
models or training data. Its main practical strength is the per-feature,
per-ROI decomposition: a human reviewer can see \emph{which} aspect of the
ball failed, rather than reading one opaque anomaly score. The results in
Table~\ref{tab:results} show this working as intended -- seam marks raise
only the seam score, heavy wear raises patch, seam, and character scores
together, and the filled/recolored characters raise character-level scores
specifically. 

\subsection{Limitations}

\begin{itemize}
\item \textbf{Fixed ROI coordinates assume consistent framing.} All ROIs
use fixed fractional coordinates of the registered crop, assuming
top-down framing similar to the reference; a steeper angle or a
differently-printed ball would misalign them even after registration. This
same rigidity also means defects outside the defined ROIs go unchecked
entirely -- the small blue/yellow fleck near the ``T'' in
\texttt{deform\_L\_N\_Symbol} falls just outside the first character's
bounding box, so its color was never sampled at all, independent of any
threshold setting.
\end{itemize}

\subsection{Possible Extensions}
Two natural directions emerge for extending this work.

\paragraph{Classical Computer Vision.}
Deriving reference color signatures from multiple clean-ball images
rather than a single one would improve robustness to lighting variation
and reduce sensitivity to the choice of reference frame.

\paragraph{Deep Learning.}
The pipeline could serve as a structured feature extractor -- feeding
it into a lightweight neural architecture
trained end-to-end to classify defect types, allowing the system to
learn discriminative thresholds directly from labeled data rather than
relying on manual calibration.

\subsection{Conclusion}

This project shows that a classical pipeline -- Hough-transform
localization, saturation-based seam registration, CIELAB color histograms
compared via Earth Mover's Distance, and morphologically-filtered
structural XOR -- reliably distinguishes clean and worn tennis balls and
correctly localizes most introduced defects to the right feature, while
remaining fully interpretable at every stage. Its main weakness, missing a
heavily faded color defect, is a direct consequence of fixed,
manually-set thresholds rather than a flaw in the underlying approach.
